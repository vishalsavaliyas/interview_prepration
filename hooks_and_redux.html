React & Redux – Complete Interview Preparation Guide

1. Core React Concepts

Props

What: Props are read-only data passed from parent to child.

Why: Used for component communication.

function Child({ name }) {
  return <h1>Hello {name}</h1>;
}

State

What: State holds mutable data that affects UI.

Why: UI re-renders when state changes.

2. React Hooks – Complete Explanation

useState

Manages simple local state.

const [count, setCount] = useState(0);

useEffect

Handles side effects like API calls.

useEffect(() => {
  fetchData();
}, []);

useContext

Shares global data without prop drilling.

useRef

Stores mutable value without re-render.

useReducer

Manages complex state logic using reducer pattern.

useMemo

Memoizes expensive calculations.

useCallback

Memoizes functions to avoid re-renders.

Other Hooks

  • useLayoutEffect
  • useImperativeHandle
  • useId
  • useTransition
  • useDeferredValue
  • Custom Hooks

3. useContext vs useReducer vs Redux

Feature useContext useReducer Redux
Scope Global sharing Local Global
Complexity Low Medium High
Async support No No Yes
DevTools No No Yes
Best for Theme/Auth Complex local logic Large apps
Interview Tip: useContext + useReducer can replace Redux for medium-sized apps.

4. Redux – In Depth Explanation

What is Redux?

Redux is a predictable state management library for JavaScript applications.

Why Redux?

  • Single source of truth
  • Predictable updates
  • Easy debugging

Core Principles

  • Single source of truth
  • State is read-only
  • Changes via pure functions

Redux Data Flow

UI → dispatch(action)
        ↓
     Reducer
        ↓
     Store
        ↓
     UI updates

Redux Building Blocks

  • Store – holds state
  • Action – describes what happened
  • Reducer – updates state
  • Dispatch – sends action

5. Redux Toolkit (Modern Redux)

Redux Toolkit is the official, recommended way to use Redux.

createSlice

const cartSlice = createSlice({
  name: "cart",
  initialState: [],
  reducers: {
    addItem(state, action) {
      state.push(action.payload);
    }
  }
});

createAsyncThunk

Handles async API calls with pending / fulfilled / rejected states.

6. When to Use What?

Scenario Best Choice
Simple UI state useState
Complex component logic useReducer
Theme / Auth useContext
Large app, shared state Redux Toolkit

7. Final Interview Statements

“useContext solves data sharing, useReducer solves state logic, and Redux solves large-scale predictable state management.”
“Redux should be used when debugging, scalability, and predictable state updates matter more than simplicity.”
