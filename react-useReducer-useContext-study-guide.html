2Ô∏è‚É£ useReducer ‚Äì HOW State Changes

When to Use

  • Multiple state values
  • Complex update logic
  • Actions like ADD / DELETE / UPDATE

Mental Model

UI ‚ûú dispatch(action) ‚ûú reducer ‚ûú new state

Basic Pattern

const initialState = { count: 0 }

function reducer(state, action) {
  switch (action.type) {
    case "INCREMENT":
      return { count: state.count + 1 }

    case "DECREMENT":
      return { count: state.count - 1 }

    default:
      return state
  }
}

const [state, dispatch] = useReducer(reducer, initialState)

Key Rules (EXAM + INTERVIEW)

  • Reducer must be pure
  • No direct state mutation
  • Always return new state
  • Actions describe events, not logic

3Ô∏è‚É£ useContext ‚Äì WHERE State Is Available

Problem It Solves

Avoids prop drilling when data is needed deeply.

Pattern

const AppContext = React.createContext()

function App() {
  return (
    <AppContext.Provider value={{ user: "Nisha" }}>
      <Home />
    </AppContext.Provider>
  )
}

function Home() {
  const data = useContext(AppContext)
  return <h2>Hello {data.user}</h2>
}

Rules

  • Context does NOT manage logic
  • Context only shares data
  • Too much context causes re-renders

Step-by-Step Flow

1. initialState ‚ûú define data
2. reducer ‚ûú define actions
3. Context ‚ûú create store
4. Provider ‚ûú pass state + dispatch
5. Components ‚ûú consume + dispatch

Code Flow

// initialState.js
export const initialState = {
  todos: [],
  user: null
}

// reducer.js
export function reducer(state, action) {
  switch (action.type) {
    case "ADD_TODO":
      return {
        ...state,
        todos: [...state.todos, action.payload]
      }

    case "LOGIN":
      return {
        ...state,
        user: action.payload
      }

    default:
      return state
  }
}

// AppContext.js
export const AppContext = createContext()

export function AppProvider({ children }) {
  const [state, dispatch] = useReducer(reducer, initialState)

  return (
    <AppContext.Provider value={{ state, dispatch }}>
      {children}
    </AppContext.Provider>
  )
}

// Any component
const { state, dispatch } = useContext(AppContext)

dispatch({ type: "ADD_TODO", payload: "Learn React" })
