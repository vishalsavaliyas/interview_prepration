React Basics — What is React? + Setup


What is React?

React is a JavaScript library for building user interfaces. It was developed by Facebook and is widely used for creating single-page applications and mobile applications. React allows developers to create reusable UI components, manage the state of their applications, and efficiently update the DOM (Document Object Model) when the state changes.

React is a JavaScript library used to build:
Web apps
UI components (button, navbar, cards)
Single Page Applications (SPA)

Why React is so popular?
✅ Fast
✅ Easy to use
✅ Component-based
✅ Huge community

React is for:
✔ Frontend
✔ UI
✔ Making screens

React is not a framework. It is a library that focuses on building UI. It can be used with other libraries or frameworks like Redux, React Router, etc.

Setting Up a React Application

To set up a React application, you can follow these steps:

  1. Install Node.js and npm (Node Package Manager) on your development environment.
  2. Create a new directory for your React application and navigate to it in your terminal.
  3. Run the following command in your terminal or in vs code terminal to create a new React app:
    npm create vite@latest my-react-app --template react
  4. Navigate to the newly created directory and run the following command to install the dependencies:
    npm install
  5. Run the following command to start the development server:
    npm run dev
  6. to exit the development server, press Ctrl + C in the terminal
  7. Open your web browser and go to http://localhost:5173 to view your React application.
  8. You can now start building your React application by editing the files in the src directory.
  9. Open: src/App.jsx

This will start a local development server, and you can view your React application in your web browser at http://localhost:5173.

Conclusion

React is a powerful JavaScript library for building user interfaces, and setting up a React application is a simple process. By following these steps, you can create a basic React application and start building your user interface.

what is vite vs CRA (Create React App)

Vite and Create React App (CRA) are both tools used to set up and develop React applications, but they have some key differences in terms of their architecture, performance, and features.

Vite

Vite is a modern build tool that leverages native ES modules in the browser for faster development and build times. It uses a dev server that serves files over native ES modules, allowing for instant module loading and hot module replacement (HMR) without the need for bundling during development. Vite also supports features like TypeScript, JSX, and CSS modules out of the box.

Vite is a modern frontend development build tool created by Evan You (creator of Vue.js). It is designed to be faster, lightweight, and more efficient than older tools.


Where is Vite used?

For creating modern JavaScript web applications
For frameworks like React, Vue, Svelte, Preact, SolidJS, Vanilla JS
Ideal for projects needing fast startup, hot reload, and optimized builds

Why developers use Vite?

Extremely fast dev server
Uses native ES modules in the browser
Lightning-fast Hot Module Replacement (HMR)
Optimized production builds using Rollup

Create React App (CRA)

Create React App is a popular tool for setting up React applications that provides a pre-configured build setup using Webpack and Babel. CRA abstracts away the configuration details, allowing developers to focus on writing code without worrying about the underlying build process. CRA includes features like hot module replacement, code splitting, and optimized production builds.

Create React App (CRA) is an older tool created by Facebook to quickly set up a React project without manual configuration.

Key Differences

  • Performance: Vite is generally faster than CRA during development due to its use of native ES modules and on-demand module loading.
  • Configuration: CRA abstracts away configuration details, while Vite allows for more customization and flexibility in the build process.
  • Ecosystem: CRA has a larger ecosystem and community support, while Vite is relatively newer but gaining popularity quickly.
  • Features: Vite provides features like hot module replacement, CSS modules, and TypeScript support, while CRA has features like code splitting and optimized production builds.

Conclusion

Both Vite and Create React App are excellent tools for setting up and developing React applications. The choice between the two ultimately depends on your specific needs and preferences as a developer. If you prioritize speed and flexibility, Vite may be the better choice. If you prefer a more opinionated setup with a larger ecosystem, CRA may be the way to go.

what is react router

React Router is a popular library for building single-page applications with React. It provides a way to manage navigation and routing within a React application, allowing developers to create dynamic and interactive user interfaces.

React Router allows developers to define routes for different pages or components within the application. When a user navigates to a specific route, React Router will render the corresponding component and update the URL in the browser's address bar.

Key Features of React Router

  • Provides a way to manage navigation and routing within a React application.
  • Allows developers to define routes for different pages or components within the application.
  • Supports dynamic routing, allowing developers to create routes that change based on user input or other factors.
  • Integrates with React's component-based architecture, making it easy to create reusable and modular components.

Conclusion

React Router is a powerful library for building single-page applications with React. It provides a way to manage navigation and routing within a React application, allowing developers to create dynamic and interactive user interfaces. With its support for dynamic routing and integration with React's component-based architecture, React Router is an essential tool for any React developer.
