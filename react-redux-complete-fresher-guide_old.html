React & Redux – Complete Guide for Freshers

1. What is React?

React is a JavaScript library used to build user interfaces. It allows us to create UI using components.

Why React?

  • Reusable components
  • Fast UI updates (Virtual DOM)
  • Easy to manage UI state

2. Components

A component is a function that returns UI. 

function Hello() {
  return <h1>Hello World</h1>;
}
Components are like LEGO blocks – small, reusable pieces.

3. Props (Properties)

What are Props?

Props are data passed from parent to child. Props are read-only. 

function Child(props) {
  return <h2>Hello {props.name}</h2>;
}

function Parent() {
  return <Child name="Nisha" />;
}
Props flow only one way: Parent → Child

4. State

What is State?

State is data that changes over time and affects UI. 

const [count, setCount] = useState(0);

When state changes → React re-renders the component.

5. React Hooks (Basics)

Hooks are functions that allow us to use React features like state and lifecycle in functional components.

Why Hooks?

  • No class components needed
  • Cleaner code
  • Reusable logic

6. useState

useState is used to store and update local state. 

const [name, setName] = useState("John");

setName("Alex");
Use useState for simple values: input, toggle, counter

7. useEffect

useEffect is used for side effects.

Examples

  • API calls
  • Timers
  • Event listeners
useEffect(() => {
  console.log("Component mounted");
}, []);
Empty dependency array → runs only once

8. useRef

useRef stores a value that does NOT cause re-render. 

const inputRef = useRef();

<input ref={inputRef} />
useRef is mainly used for DOM access

9. useContext

useContext is used to share data globally without prop drilling. 

const ThemeContext = createContext();

<ThemeContext.Provider value="dark">
  <App />
</ThemeContext.Provider>
  
const theme = useContext(ThemeContext);
Best for theme, auth user, language

10. useReducer

useReducer is used when state logic is complex. 

function reducer(state, action) {
  switch(action.type) {
    case "INC":
      return { count: state.count + 1 };
    default:
      return state;
  }
}

const [state, dispatch] = useReducer(reducer, { count: 0 });
useReducer is like small Redux inside component

11. Redux (In Simple Words)

Redux is used to manage global application state.

Why Redux?

  • Large applications
  • Same data used in many components
  • Better debugging

12. Redux Core Concepts

Store

Holds the entire app state.

Action

Plain object that describes what happened.

{ type: "ADD_TO_CART", payload: item }

Reducer

Pure function that returns new state.

Dispatch

Sends action to reducer.

13. Redux Data Flow

User Action
   ↓
dispatch(action)
   ↓
reducer
   ↓
store updated
   ↓
UI re-render

14. Redux Toolkit

Redux Toolkit is the modern and recommended way to use Redux. 

const cartSlice = createSlice({
  name: "cart",
  initialState: [],
  reducers: {
    addItem(state, action) {
      state.push(action.payload);
    }
  }
});
Redux Toolkit uses Immer internally (safe mutation)

15. useContext vs useReducer vs Redux

Feature useContext useReducer Redux
Scope Global sharing Local Global
Complexity Low Medium High
Best for Theme/Auth Complex logic Large apps

16. Final Interview Memory Lines

useState → simple local state
useContext → avoid prop drilling
useReducer → complex state logic
Redux → large app global state
