1. Closure ‚≠ê

A closure allows a function to access variables from its outer scope even after the outer function has finished executing.

function outer() {
  let count = 0;
  return function inner() {
    count++;
    return count;
  }
}
const counter = outer();
counter(); // 1
counter(); // 2

2. Hoisting

JavaScript moves variable and function declarations to the top of their scope.

console.log(a); // undefined
var a = 10;

3. Event Loop ‚≠ê

The event loop handles asynchronous operations like setTimeout, promises, and API calls.

console.log("A");
setTimeout(() => console.log("B"), 0);
console.log("C");
// Output: A C B

4. Call Stack & Memory Heap

5. Synchronous vs Asynchronous

Sync ‚Üí Blocks execution

6. Debouncing ‚≠ê

Limits function execution after a delay.

function debounce(fn, delay) {
  let timer;
  return function () {
    clearTimeout(timer);
    timer = setTimeout(fn, delay);
  }
}

7. Throttling ‚≠ê

Executes function at fixed intervals.

function throttle(fn, limit) {
  let flag = true;
  return function () {
    if (flag) {
      fn();
      flag = false;
      setTimeout(() => flag = true, limit);
    }
  }
}

8. Currying

Breaking a function with multiple arguments into single-argument functions.

const add = a => b => c => a + b + c;
add(1)(2)(3); // 6

9. Memoization

Optimizes performance by caching results.
