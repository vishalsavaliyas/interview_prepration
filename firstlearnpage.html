Setting Up a React Application (Using Vite)

  1. Install Node.js from the official site.
  2. Open Terminal or VS Code Terminal.
  3. Create React app using Vite:
    npm create vite@latest my-react-app --template react
  4. Navigate into folder:
    cd my-react-app
  5. Install dependencies:
    npm install
  6. Start server:
    npm run dev
  7. Visit your app at http://localhost:5173

Vite vs Create React App (CRA)

Both are tools to create React apps, but Vite is the newer, faster alternative.

Vite

  • Very fast startup
  • Uses native ES modules
  • Instant Hot Module Reload (HMR)
  • Lightweight & modern

Create React App (CRA)

  • Based on Webpack
  • Slower startup
  • Good for older tutorials
  • No longer recommended officially

Key Differences

  • Speed: Vite is much faster.
  • Build system: Vite uses modern Rollup; CRA uses Webpack.
  • Community: CRA has older content; Vite is the current standard.

What is JSX?

JSX (JavaScript XML) lets you write HTML-like syntax inside JavaScript. Browsers can't read JSX, so it is compiled into JavaScript.

Example: JSX vs JavaScript

// Using JSX
const element = <h1>Hello JSX!</h1>;

// Without JSX
const element = React.createElement("h1", null, "Hello JSX!");

What is the React Compiler?

The React Compiler (formerly Babel + React tooling) converts JSX into plain JavaScript so browsers can run it.

  • Converts JSX â†’ JavaScript
  • Optimizes React code
  • Improves performance

What is React Router?

React Router lets you create multi-page navigation inside a single-page application.

  • Handles page navigation
  • Supports dynamic routing
  • Changes URL without reloading page

Example: Basic React Router Setup

import { BrowserRouter, Routes, Route } from "react-router-dom";
import Home from "./Home";
import About from "./About";

export default function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}

React Forms & Controlled Components

useRef Hook

useRef is used to access DOM elements or store values without re-rendering.

Example

import { useRef } from "react";

export default function App() {
  const inputRef = useRef();

  function focusInput() {
    inputRef.current.focus();
  }

  return (
    <div>
      <input ref={inputRef} type="text" />
      <button onClick={focusInput}>Focus Input</button>
    </div>
  );
}

useReducer Hook

useReducer is used for complex state logic.

Example

function reducer(state, action) {
  switch (action.type) {
    case "increment":
      return { count: state.count + 1 };
    default:
      return state;
  }
}

export default function Counter() {
  const [state, dispatch] = useReducer(reducer, { count: 0 });

  return (
    <div>
      <h2>Count: {state.count}</h2>
      <button onClick={() => dispatch({ type: "increment" })}>+</button>
    </div>
  );
}

Custom Hooks

You can create your own hooks for reusable logic.

Example

function useCounter() {
  const [count, setCount] = useState(0);
  return { count, increment: () => setCount(count + 1) };
}

export default function App() {
  const { count, increment } = useCounter();
  return (
    <div>
      <h2>Count: {count}</h2>
      <button onClick={increment}>+</button>
    </div>
  );
}

Redux Toolkit (Quick Guide)

Redux Toolkit (RTK) is the recommended way to write Redux logic. It simplifies store setup, reducers, and async logic.

Example: Basic RTK setup

// store.js
import { configureStore } from "@reduxjs/toolkit";
import counterReducer from "./counterSlice";

export const store = configureStore({
  reducer: { counter: counterReducer }
});

// counterSlice.js
import { createSlice } from "@reduxjs/toolkit";

const counterSlice = createSlice({
  name: "counter",
  initialState: { value: 0 },
  reducers: {
    increment: state => { state.value += 1 },
    decrement: state => { state.value -= 1 }
  }
});

export const { increment, decrement } = counterSlice.actions;
export default counterSlice.reducer;

React Router â€” Full Tutorial (extra features)

Extended topics: nested routes, route parameters, protected routes, lazy loading routes.

Example: Protected Route & Lazy loading

import { BrowserRouter, Routes, Route, Navigate } from "react-router-dom";
import { lazy, Suspense } from "react";

const Home = lazy(() => import("./pages/Home"));
const Dashboard = lazy(() => import("./pages/Dashboard"));

function Protected({ children, isAuth }) {
  return isAuth ? children : <Navigate to="/login" />;
}

export default function App() {
  const isAuth = false; // replace with real auth state
  return (
    <BrowserRouter>
      <Suspense fallback={<div>Loading...</div>}>
        <Routes>
          <Route path="/" element={<Home />} />
          <Route path="/dashboard" element={<Protected isAuth={isAuth}><Dashboard /></Protected>} />
        </Routes>
      </Suspense>
    </BrowserRouter>
  );
}

Authentication (JWT & Firebase)

JWT (JSON Web Token)

Common for custom backend auth: server issues token, client stores (httpOnly cookie preferred), include token in Authorization header for protected API calls.

Client-side: send token

fetch('/api/protected', {
  headers: { 'Authorization': 'Bearer ' + token }
});

Firebase Authentication

Firebase provides email/password, Google, GitHub and more with easy SDKs.

Example: Firebase email signup

import { getAuth, createUserWithEmailAndPassword } from "firebase/auth";

const auth = getAuth(app);
createUserWithEmailAndPassword(auth, email, password)
  .then(userCredential => console.log(userCredential.user))
  .catch(err => console.error(err));

React + Node.js â€” Full-stack Overview

Typical architecture: React frontend (Vite) communicates with Node/Express backend via REST or GraphQL. Use CORS, env variables, and authentication.

Tips
