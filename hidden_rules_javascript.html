JavaScript Interview & Mastery Guide (2026 Edition)

1. Comparison & Type Coercion

Loose (==) vs. Strict (===) Equality

Rule: == checks Value only (allows coercion); === checks Value AND Type (strict).

Hidden Truth: NaN is the only value not equal to itself (NaN === NaN is false). Always use Number.isNaN().

5 == "5";   // true (String "5" becomes Number 5)
5 === "5";  // false (Different types)
null == undefined; // true (Common interview trick)
null === undefined; // false

2. Variables, Scope & Hoisting

The Temporal Dead Zone (TDZ)

Rule: var is hoisted and initialized as undefined. let and const are hoisted but stay in the TDZ until their declaration line.

Interview Answer: "Accessing let/const before declaration throws a ReferenceError due to the TDZ, unlike var which just gives undefined."
console.log(a); // undefined
var a = 10;

console.log(b); // ReferenceError (TDZ)
let b = 20;

Lexical "this"

Rule: Regular functions determine this by how they are called. Arrow functions inherit this from their surrounding parent scope.

3. Asynchronous Execution

The Event Loop Hierarchy

Rule: Synchronous code executes first, followed by the Microtask Queue, then the Macrotask Queue.

  • Microtasks: Promises, queueMicrotask, MutationObserver.
  • Macrotasks: setTimeout, setInterval, setImmediate.
console.log("1");
setTimeout(() => console.log("2"), 0); 
Promise.resolve().then(() => console.log("3"));
console.log("4");
// Output: 1, 4, 3, 2 (Microtask "3" runs before Macrotask "2")

4. Modern JS Features (ES2025 - ES2026)

Explicit Resource Management (using)

New in 2026: The using keyword automatically cleans up resources (database connections, file handles) when they exit the block scope.

async function processData() {
    await using connection = await db.connect(); 
    // Connection closes automatically here!
}

Records & Tuples

Concept: Immutable objects (#{}) and arrays (#[]) that are compared by Value instead of reference.

#{ a: 1 } === #{ a: 1 }; // true (Deep equality by default!)

5. Advanced Performance Rules

Deep Cloning vs. Shallow Copy

Standard: Use structuredClone(obj) for true deep copies. Avoid JSON.parse(JSON.stringify(obj)) as it destroys Dates, Sets, and Functions.

WeakMap & Garbage Collection

Rule: WeakMap keys must be objects. If the object key is deleted elsewhere, the entry is automatically removed from the WeakMap, preventing memory leaks.
